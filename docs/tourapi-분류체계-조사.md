# TourAPI 분류체계 조사 보고서

> 조사일: 2026-07-19 · 대상: 한국관광공사 국문 관광정보 서비스 (KorService2) · 인구감소지역 89곳 전수 조사 (13,189건)

## 요약 (TL;DR)

1. **카테고리 코드 있음. 단 두 체계가 공존** — 구분류 `cat1/2/3`(예: A01>A0101>A01011200 해수욕장)과 신분류 `lclsSystm1/2/3`(예: NA>NA02>NA020900 해변·해수욕장)은 **서로 다른 분류체계**다. 구분류는 v4.4 스펙에서 삭제됐고 실측 채움률 61.7%(신규 콘텐츠 위주 누락)라 신뢰 불가 → **신분류 `lclsSystm1/2/3` 기준으로 설계할 것**
2. **테마(연인/가족/체험/뚜벅이) 직접 매칭은 불가.** 분류가 시설 유형 중심이라 `소분류(240개) → 자체 테마` 매핑 테이블 필요. 소분류가 세밀해서("글램핑장", "둘레길", "카페" 수준) 충분히 가능
3. **89개 지역 카테고리 채움률 100.0%** (대·중·소 모두, 13,189건 전수 확인). 카테고리 누락 걱정은 없음. 진짜 변수는 **지역별 콘텐츠 양 편차** (최소 40건 ~ 최대 427건)
4. **연관관광지 API(TarRlteTarService1)도 89곳 전부 데이터 있음** (연관쌍 57,173건, 지역당 기준관광지 중앙값 19곳). 단 지역 코드 체계가 KorService2와 달라 매핑 테이블 필요 — 상세는 5장

> 지역별 원본 수치는 [`tourapi-조사데이터.json`](tourapi-조사데이터.json) 참고 (양쪽 API의 지역 코드 매핑 포함)

## 1. API 기본 정보

- **엔드포인트**: `https://apis.data.go.kr/B551011/KorService2`
- **주요 오퍼레이션**: `areaBasedList2`(지역기반 목록), `ldongCode2`(법정동 코드), `lclsSystmCode2`(분류체계 코드), `detailCommon2`(상세)
- **공통 파라미터**: `serviceKey`, `MobileOS=ETC`, `MobileApp=<앱명>`, `_type=json`
- **지역 필터**: `lDongRegnCd`(법정동 시도) + `lDongSignguCd`(법정동 시군구) — v4.3부터 추가. 구 `areaCode/sigunguCode`는 응답에서 빈 값
- **분류 필터**: `lclsSystm1`(대분류만) / `+lclsSystm2` / `+lclsSystm3` 조합 가능
- 전국 총 콘텐츠 50,654건 중 인구감소지역 89곳이 13,189건(26%)

### 주의사항

- v4.4(2026-02)에서 `cat1/2/3`, `areaCode`, `contentTypeId` 필터 등 구 스펙 다수 삭제됨 — 구버전 블로그/예제 코드 참고 금지
- 응답에는 `cat1/2/3` 값이 여전히 내려오는 콘텐츠가 있으나(89개 지역 실측 61.7%), 2026년 신규 등록 콘텐츠 등 38%가 빈 값. 반면 `lclsSystm`은 100% 채움 — cat은 로직에 사용하지 말 것
- API 시도 코드 기준으로 **광주+전남이 "전남광주통합특별시"(코드 12)로 통합**되어 있음. 행정구역 매핑 시 주의
- 강원·전북은 특별자치도 명칭 사용

## 4. 연관관광지 API 검증 (TarRlteTarService1)

이동 빅데이터 기반 "함께 방문한 관광지" API. 코스(동선) 구성의 뼈대로 사용 가능 여부를 89곳 전수 검증함 (기준년월 202605).

**결과: 전 지역 사용 가능**

- 89/89 지역 모두 데이터 존재 (0건 지역 없음)
- 지역당 기준(앵커) 관광지: 최소 5곳(대구 서구) / 중앙값 19곳 / 최대 78곳
- 연관쌍 총 57,173건, 그중 **25%가 타 시군구 연관** → "인접지역 포함" 로직에 자연스럽게 활용 가능

**통합 시 주의사항 (중요)**

1. **지역 코드 체계가 KorService2와 다름** — 이 API는 강원 51·전북 52(특별자치도 신코드), 군위군=대구(27720), 전남=구코드(46, 광주 통합 미반영) 스냅샷 사용. KorService2 법정동 코드와 1:1 매핑 테이블 필요 (`tourapi-조사데이터.json`에 89곳 매핑 포함)
2. **관광지 식별자가 자체 해시**(`tAtsCd`) — KorService2의 `contentid`와 직접 연결 불가. 상세정보(이미지·주소·좌표)를 붙이려면 **이름 기반 매칭** 필요
3. **연관 카테고리는 관광지(50%)/음식(41%)/숙박(9%) 3종뿐** — 체험·레저 등 세부 분류 없음. 테마 태깅은 KorService2 lclsSystm 쪽에서 수행하고, 이 API는 동선 구성용으로만 사용할 것
4. `baseYm`(기준년월) 필수 파라미터 — 월 단위 갱신이므로 최신 월 조회 로직 필요
